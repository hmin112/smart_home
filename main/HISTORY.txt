# 스마트 홈 제어 시스템 프로젝트 진행 이력 (HISTORY.txt)

## 1. 프로젝트 개요
- 목표: 라즈베리파이(웹 서버)와 아두이노(센서/액추에이터)를 연동한 통합 스마트 홈 시스템 구축
- 주요 기능: 도어락 제어, 전등 제어, 실시간 온습도 모니터링, 쓰레기통 포화도 측정, 냉난방기 제어, 전력 사용량 통계

## 2. 현재 폴더 및 파일 구조
- / (Root): 프로젝트 전체 관리 및 설정 (iot.txt: 요구사항 정의서)
- /main: Node.js 백엔드 서버 (server.js, package.json, setup_db.sh)
- /arduino: 아두이노 펌웨어 소스 (1.ino: 전등/온습도, 2.ino: 초음파/IR)
- /web: React/Vite 기반 프론트엔드 대시보드 (App.jsx, Tailwind CSS)

## 3. 진행 상황 및 구현 기능 (2026-06-06 기준)
- [백엔드]: Node.js + Express + Socket.io 서버 구축, MySQL DB 연동, 시리얼 통신 연동 완료.
- [프론트엔드]: 애플 위젯 스타일의 UI 구현, 실시간 센서 데이터(DHT11, 초음파) 시각화, 전력 사용량 계산 로직 포함.
- [아두이노 1]: DHT11 온습도 데이터 전송 및 서보모터 4개(전등 1, 2) 제어 로직 구현.
- [아두이노 2]: 초음파 센서(거리 측정) 및 KY-005 IR 송신기를 통한 에어컨 제어 구현.
- [데이터베이스]: `smarthome` DB 및 `device_logs` 테이블 자동 설치 스크립트 작성.

## 4. 기술적 어려움 및 해결 과정 (Troubleshooting)
1. **서보모터 오작동 이슈**:
   - 증상: 모터가 360도 계속 회전하거나, 하나를 꽂으면 나머지가 멈추는 현상 발생.
   - 분석: 
     - 180도 표준 서보 대신 360도 연속 회전 서보가 혼용되었을 가능성 확인.
     - 아두이노 USB 전원(500mA)만으로는 서보 4개의 기동 전류를 감당하지 못해 전압 강하(Brownout) 발생.
   - 해결(진행 중): 모터 종류 확인(MG90S 등 금속 기어 권장), 외부 전원(5V 2A 이상) 공급 및 GND 공유 필요성 확인.

2. **시리얼 포트 인식**:
   - 라즈베리파이 환경에 따라 `/dev/ttyACM0` 또는 `ttyUSB0` 경로가 유동적인 문제. 서버에서 에러 핸들링 및 Mocking 로직 추가.

3. **하드웨어 선정 고민**:
   - SG90의 힘 부족 해결을 위해 MG996R, 스테퍼 모터(28BYJ-48) 등을 검토했으나, 전력 및 핀 개수 문제로 MG90S + 외부 전원 조합이 가장 적합함을 도출.

## 7. 재부팅 후 서비스 복구 및 자동화 개선 (2026-06-09)
1. **데이터베이스 서비스 복구**
   - 원인: Raspberry Pi OS(Trixie) 환경에서 `mysql-server` 패키지 부재로 인해 DB 설치 및 실행 실패.
   - 해결: `mariadb-server` 설치 및 데이터베이스/사용자 재설정.
   - 자동화: `setup_db.sh` 스크립트를 MariaDB 기준으로 업데이트.

2. **시리얼 포트 스왑 문제 해결 (Auto-Detection)**
   - 원인: 재부팅 시 아두이노 포트 경로(`/dev/ttyACM0`, `/dev/ttyACM1`)가 무작위로 변경되어 백엔드에서 오인식 발생.
   - 해결: `server.js`의 시리얼 로직을 수정하여 수신 데이터 패턴(T: 온습도, D: 거리)에 따라 Arduino 1/2를 자동으로 식별하도록 개선.
   - 결과: 포트 순서가 바뀌어도 별도 설정 없이 정상 작동함.

3. **서비스 의존성 최적화**
   - `smarthome-backend.service` 설정에서 `mysql.service` 의존성을 `mariadb.service`로 수정하여 부팅 시 안정적인 시작 보장.

4. **현 상태 점검**
   - 백엔드: MariaDB 연결 및 아두이노 2대 인식 완료 (3001 포트).
   - 프론트엔드: 정상 작동 확인 (5173 포트).
   - 도어락: pinctrl 기반 정상 작동 유지.

## 6. 추가 작업 이력 (2026-06-06 세션)
### 자동 시작 및 연결 설정 개선
1. **백엔드/프런트엔드 자동 시작 설정 (systemd)**
   - 라즈베리 파이 재부팅 시 서버가 자동으로 실행되지 않는 문제 해결.
   - `smarthome-backend.service` 및 `smarthome-frontend.service` 생성 및 등록.
   - 사용 명령어:
     - `sudo systemctl daemon-reload`
     - `sudo systemctl enable smarthome-backend smarthome-frontend`
     - `sudo systemctl start smarthome-backend smarthome-frontend`

2. **프런트엔드 연결 설정 동적화**
   - `App.jsx`에 하드코딩된 `localhost` 및 특정 IP 주소를 `window.location.hostname`을 사용하여 접속 환경에 맞게 자동으로 설정되도록 수정.
   - 외부 기기(폰, 노트북 등)에서 접속 시 백엔드 연결이 끊기는 문제 해결.

3. **백엔드 시작 스크립트 추가**
   - `package.json`에 `"start": "node server.js"` 스크립트를 추가하여 서비스 실행 표준화.

### 센서 데이터 및 시스템 점검
1. **온습도 센서(DHT11) 정상 작동 확인**
   - 백엔드 로그 및 시리얼 통신 직접 확인을 통해 데이터 수신 검증.
   - 사용 명령어:
     - `sudo journalctl -u smarthome-backend -f` (실시간 로그 확인)
     - `stty -F /dev/ttyACM0 9600 raw -echo; head -n 10 /dev/ttyACM0` (시리얼 원천 데이터 확인)
   - 확인된 데이터: 온도 약 25.3°C, 습도 약 58~64% 등 정상 범위.

2. **시스템 상태 점검**
   - `hostname -I`를 통한 IP 주소(`172.30.1.27`) 확인 및 프런트엔드 포트(`5174`) 확인 완료.

### 에너지 소비 분석 및 지능형 가이드 시스템 구축 (2026-06-13)
1. **에너지 어드바이저(Energy Advisor) 기능 구현**
   - **날씨 기반 벤치마킹:** 단기 프로젝트 특성상 부족한 과거 데이터를 보완하기 위해 Open-Meteo API를 연동. 외부 기온에 따른 권장 사용량을 실시간으로 계산하여 현재 사용량과 비교하는 지능형 분석 로직 도입.
   - **월간 요금 예측:** 현재까지의 사용 추세를 바탕으로 월말 예상 전기 요금을 계산하고, 사용자가 설정한 목표 금액과 비교하여 '안전/주의/위험' 상태 판별.
   - **지능형 가이드:** 분석 결과에 따라 "에어컨 설정 온도 조절 권장" 등 실질적인 에너지 절약 가이드를 텍스트로 제공.

2. **사용자 인터페이스(UI) 고도화 및 레이아웃 최적화**
   - **모달(Modal) 기반 통합:** 대시보드 공간 효율성을 위해 '에너지 어드바이저'를 별도 카드에서 '전력 사용량' 패널 내 버튼 클릭 시 나타나는 모달 팝업 형태로 전환.
   - **레이아웃 재구성:** 사용자 피드백을 반영하여 '전등'과 '실시간 실내 온습도' 패널의 위치를 스왑(Swap)하고, 전체적인 모니터링/제어 위젯 배치를 최적화.
   - **UI 디테일 개선:** 모달 내 데이터 정렬(2열 그리드), 상태 배지 위치 조정, 불필요한 텍스트 정리 등 시각적 완성도 향상.

3. **시스템 안정성 및 데이터 정밀도 개선 (Troubleshooting)**
   - **DB JSON 파싱 이슈 해결:** MySQL의 JSON 컬럼 데이터를 백엔드에서 처리할 때 발생하던 타입 오류(undefined)를 방지하기 위해 이중 파싱 및 Fallback 로직 적용.
   - **좀비 프로세스 해결:** 백그라운드에 남아있던 구형 서버 프로세스가 신규 코드를 간섭하던 문제를 식별하고, 프로세스 강제 종료 후 Node.js 직접 실행 방식으로 전환하여 데이터 연동 정상화.
   - **프론트엔드 방어 코드:** API 응답 지연이나 데이터 누락 시 화면이 하얗게 변하는 현상을 방지하기 위해 React 컴포넌트 내 Optional Chaining 및 예외 처리 강화.

4. **데이터베이스 확장**
   - `settings` 테이블 신규 생성: 사용자의 목표 전기 요금 설정을 영구적으로 저장하고 관리할 수 있도록 스키마 확장 및 연동 완료.

### 기기 상태 영구 보존 및 날씨 연동 기능 강화 (2026-06-13 추가)
1. **기기 상태 보존 (State Persistence) 구현**
   - **문제:** 웹 페이지를 새로고침하면 전등과 냉난방기(온도/모드)가 모두 초기화되는 현상.
   - **해결:** 사용자가 기기를 제어할 때마다 백엔드의 `deviceStates` 객체를 업데이트하고, 이를 DB의 `settings` 테이블에 실시간으로 JSON 형태로 저장. 프론트엔드는 접속 시 `/api/device-states`를 호출하여 이전 상태를 복구하도록 구현.
   - **온도 보정 로직:** 냉방/난방 모드 전환 시, 각 모드의 유효 온도 범위(냉방 18~27도, 난방 23~30도)를 벗어나지 않도록 자동 보정 로직(`Math.min/max`) 추가.

2. **날씨 맞춤형 알약 가이드 (WeatherPill) 추가**
   - 대시보드 상단에 실시간 날씨와 습도를 기반으로 기기 제어를 추천하는 인터랙티브 위젯 구현.
   - **5가지 시나리오 적용:** 비/습함(제습 추천), 폭염(터보 냉방), 한파(난방 추천), 애매한 습도(절전 제습), 완벽한 날씨(전원 OFF/환기).
   - **원클릭 제어:** 가이드의 '적용하기' 버튼 클릭 시 추천 설정이 실제 냉난방기 상태 및 하드웨어(Socket.io)로 즉시 전송됨. 우측 [X] 버튼으로 즉시 가동 중지 가능.

3. **날씨 정보 통합 및 동기화 (Weather API Synchronization)**
   - **아이콘 불일치 해결:** 메인 날씨 위젯, 날씨 상세 모달의 '지금' 시간대, 상단 알약 가이드가 서로 다른 API 파라미터(`current_weather` vs `current`)를 사용하여 아이콘이 불일치하는 버그 수정.
   - **해결 방식:** 모든 컴포넌트가 `current=temperature_2m,relative_humidity_2m,weather_code` 배열 형식의 API 호출로 통일하여 완벽한 데이터 동기화 확보.
     - **날씨 모달 강화:** 날씨 위젯 클릭 시 체감 온도, 실외 습도, 풍속과 함께 **24시간 시간대별 날씨 예보(수평 스크롤)**를 제공하는 기능 추가.

   ### 프로젝트 통합 관리 및 지능형 자동화 고도화 (2026-06-13 후반)
   1. **워크스페이스 통합 관리 환경 구축**
      - **루트 package.json 도입:** 프론트엔드(`web`)와 백엔드(`main`)를 루트 디렉토리에서 한 번에 제어할 수 있도록 통합 설정 파일을 생성했습니다.
      - **명령어 단축:** `npm run dev`(웹 실행), `npm start`(백엔드 실행), `npm run install:all` 등의 스크립트를 통해 개발 생산성을 향상시켰습니다.

   2. **지능형 자동화(Automation) 로직 정밀화**
      - **임계값 최적화:** 여름(냉방) 시작 온도를 **27도**, 겨울(난방) 시작 온도를 **22도**로 조정하여 실제 거주 쾌적도를 높였습니다.
      - **즉각적 반응성 확보:** 자동화 모드를 활성화하거나 목표 온도를 변경하는 즉시 에어컨이 반응하도록 `lastKnownTemp` 기반의 트리거 로직을 구현했습니다.
      - **상태 보존(Persistence):** 자동화 ON/OFF 상태와 목표 온도가 DB에 실시간 저장되어, 페이지를 새로고침하거나 서버를 재시작해도 이전 설정이 완벽히 유지되도록 개선했습니다.

   3. **사용자 경험(UX) 및 인터페이스 혁신**
      - **슬라이더 기반 온도 조절:** 기존의 수동 버튼(+/-)을 제거하고, '자동화 모드' 텍스트 클릭 시 나타나는 **슬라이더형 모달창**으로 인터페이스를 일원화하여 조작 편의성을 극대화했습니다.
      - **UI 디테일 보정:** 자동화 토글 버튼의 원이 밖으로 튀어나가던 렌더링 오류를 수정하고, 메인 위젯의 목표 온도 표시를 간결하게 정리했습니다.

   4. **에너지 어드바이저 밸런스 정밀 튜닝**
      - **현실적인 효율 계산:** 에어컨을 26도로 틀었을 때 효율이 너무 낮게 나오던 문제를 해결하기 위해 권장 전력량(Benchmark) 수치를 재보정했습니다.

      - **DB 스키마 보완:** `setup_db.sh`에 누락되었던 `settings` 테이블 생성 구문을 추가하여 초기 설치 시의 결함을 제거했습니다.
      - **코드 정형화:** 프론트엔드 코드의 ESLint 오류(미사용 변수, useEffect cascading render 등)를 전면 수정하여 빌드 안정성을 확보했습니다.
      - **좀비 프로세스 정리:** 포트 충돌(EADDRINUSE)을 유발하던 중복 서버 프로세스들을 정리하고 구동 환경을 최적화했습니다.

### 머신러닝 기반 AI 에너지 코치(AI Energy Coach) 도입 (2026-06-15)
1. **머신러닝 예측 모델 개발 (Python/scikit-learn)**
   - **모델 선정:** 실제 전력 소비 패턴을 학습하기 위해 **Random Forest Regressor** 모델을 도입했습니다.
   - **학습 로직:** `device_logs` 테이블의 기기 ON/OFF 데이터를 일별 사용 시간(Hours)으로 가공하고, 기온/습도와의 상관관계를 분석하여 미래 소비량을 예측합니다.
   - **데이터 보완:** 초기 도입 시 데이터 부족 문제를 해결하기 위해, 사용자의 기기 구성(전등 10W, 에어컨 1.5kW 등)을 기반으로 한 **합성 데이터 생성(Synthetic Data Generation)** 로직을 포함하여 즉시 예측이 가능하도록 구현했습니다.
   - **신뢰도 지표:** 모델의 결정계수(R²)를 기반으로 'AI 예측 신뢰도'를 산출하여 사용자에게 데이터의 타당성을 제공합니다.

   - **엔드포인트:** `GET /api/ai-energy-coach` 신규 추가.
   - **연동 방식:** Node.js의 `child_process.exec`를 사용하여 파이썬 학습 스크립트를 비동기적으로 실행하고, 결과를 JSON으로 파싱하여 프론트엔드에 전달합니다.
   - **의존성 설치:** `scikit-learn`, `pandas`, `pymysql`, `numpy` 등 머신러닝 필수 라이브러리를 라즈베리 파이 시스템에 구축했습니다.
